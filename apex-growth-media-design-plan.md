# Apex Growth Media — Design Plan & Site Content

## 1. Design Direction (the "why")

Sheryians works because everything ties back to one feeling: **momentum** — you're behind, this gets you ahead, fast. For Apex Growth Media, the equivalent feeling isn't "get placed," it's **growth you can watch happen** — leads climbing, ROAS climbing, followers climbing. So instead of borrowing Sheryians' visual skin, we borrow its _confidence_ and build the whole identity around an ascending/ticking motif: charts going up, numbers counting up, a live-feed energy.

**Signature element:** A horizontal **"Live Growth Ticker"** — a thin animated strip (like a stock ticker) that scrolls real client-style metrics continuously: `↑ 312% Leads · ↑ 4.8x ROAS · ↑ 89 Campaigns Live · ↑ 2.1Cr Managed Spend`. It sits right under the hero on every page in a slightly different context (homepage: overall brand proof; service pages: service-specific metric; case studies: that client's numbers). This is the one thing a visitor remembers — nobody else in this space uses a persistent live-metric spine instead of static stat boxes.

## 2. Color Palette

| Name            | Hex       | Use                                                                                            |
| --------------- | --------- | ---------------------------------------------------------------------------------------------- |
| Void Ink        | `#0B0E12` | Primary background (near-black, not pure black — has a blue undertone)                         |
| Slate Fog       | `#161B22` | Card / section surfaces, raised panels                                                         |
| Signal Green    | `#3ADE7A` | "Proof" accent — stats, tickers, checkmarks, anything that says "this worked"                  |
| Ember Coral     | `#FF5A3C` | Action accent — all CTAs, buttons, links. Warm, urgent, never used for anything passive        |
| Electric Indigo | `#6C63FF` | Secondary accent — reserved only for the AI/automation service, so it reads as "the smart one" |
| Cloud White     | `#F5F7FA` | Primary text on dark backgrounds                                                               |
| Ash Grey        | `#8B94A3` | Secondary/muted text, captions                                                                 |

**Rule:** Green = proof/results. Coral = action/CTA. Indigo = AI features only. Never mix the accent roles — that discipline is what keeps it from looking like generic "dark mode + one neon color" template.

---

## 3. Typography

| Role                | Typeface                | Notes                                                                                                                               |
| ------------------- | ----------------------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| Display (headlines) | **Bricolage Grotesque** | Has a slightly irregular, hand-adjusted geometric feel — confident without being a cold corporate grotesk. Use Bold/ExtraBold only. |
| Body                | **Inter**               | Clean, highly legible at small sizes, pairs quietly against Bricolage's personality                                                 |
| Data/Utility        | **JetBrains Mono**      | Used ONLY for numbers — stats, prices, the ticker, percentages. Monospace numbers reinforce "we measure everything"                 |

Type scale suggestion: Hero H1 ~64px/Bold, Section H2 ~40px/Bold, Body ~17px/Regular, Ticker/stat numbers ~28px/Mono Bold.

## 6. Motion Notes

- Ticker: continuous horizontal scroll, pause on hover
- Hero visual: numbers/bars animate upward on page load (once, not looping — avoid the "everything pulses forever" AI-generated feel)
- Cards: subtle lift + border-glow (in the card's assigned accent color) on hover only
- Respect `prefers-reduced-motion` — freeze ticker and skip animations for users who request it

---

## 7. Build Note

When you get to code: use shadcn/Tailwind for structural primitives (forms, accordions, carousels) but override all default styling with this palette/type system — don't ship shadcn's default look. That's what will make it feel like Apex, not "a shadcn site."
